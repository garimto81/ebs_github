## 개요

EBS 3-앱 아키텍처의 비기능 요구사항(Non-Functional Requirements)을 정의한다.

## 1. 성능

| 지표 | Phase 1 목표 | Phase 3 목표 | 측정 방법 |
|------|:----------:|:----------:|----------|
| RFID 카드 → 오버레이 표시 | < 200ms | < 100ms | E2E 타임스탬프 |
| CC 액션 → 오버레이 반영 | < 150ms | < 80ms | WebSocket 왕복 |
| Lobby 페이지 로드 (LCP) | < 2s | < 1s | Lighthouse |
| API 응답 시간 (p95) | < 500ms | < 200ms | 서버 로그 |
| WebSocket 메시지 처리 | < 50ms | < 20ms | 클라이언트 로그 |

## 2. 가용성

| 지표 | 목표 | 비고 |
|------|------|------|
| 연속 운영 | ≥ 4시간 (Phase 1) / ≥ 12시간 (Phase 3) | 무중단 방송 |
| 크래시 복구 | < 30초 자동 재시작 | 세션 복원 (BS-02) |

## 3. 동시 접속

| 컴포넌트 | Phase 1 | Phase 3 |
|----------|:-------:|:-------:|
| 동시 테이블 (CC+Overlay) | 3 | 12 |
| Lobby 동시 사용자 | 5 | 20 |
| WebSocket 동시 연결 | 10 | 50 |

## 4. 데이터

| 지표 | 요구사항 |
|------|---------|
| RFID 인식률 | ≥ 99.5% (52장 기준) |
| 데이터 보존 | 핸드 기록 최소 1년 |
| 백업 주기 | 매 방송 종료 후 자동 |
