# 3. Change Requests — DEPRECATED (2026-04-17)

> **이 프로세스는 v7 거버넌스 (`free_write_with_decision_owner`) 도입으로 완전 폐기되었습니다.**
> 현재 문서 변경 방식: 자유 편집 + decision_owner 판정. 상세: `../../CLAUDE.md` §"문서 변경 거버넌스".
>
> 아래 내용은 역사 기록으로 보존합��다.

---

**(아래는 폐기 전 역사 기록)**

팀이 계약(다른 팀 소유 문서, Shared 등)을 변경 제안하던 경로였음.

## 라이프사이클

```
1. pending/      (팀이 draft 작성)       파일명: CR-teamN-YYYYMMDD[-slug].md
       ↓
2. in-progress/  (Conductor 가 승격)     파일명: CR-NNN-{slug}.md
       ↓
3. done/         (처리 완료, archived)   원본 draft + 승격본
```

## 폴더별 편집 권한

| 폴더 | 편집 가능 | 파일명 패턴 |
|------|-----------|-------------|
| `pending/` | 제안 팀만 (자기 팀 prefix) | `CR-teamN-YYYYMMDD[-slug].md` |
| `in-progress/` | Conductor 만 (체크리스트 수정) | `CR-NNN-{slug}.md` |
| `done/` | read-only | — |

## 리스크 분류 (Fast-Track)

| 등급 | 조건 | 처리 경로 |
|------|------|----------|
| LOW | 추가 전용, 영향팀 ≤1 | publisher 직접 반영 + 영향팀 1명 approve |
| MEDIUM | 비파괴 수정, 영향팀 ≤2 | publisher 직접 반영 + 영향팀 전원 approve |
| HIGH | 파괴적 변경, 영향팀 3+ | 풀 CR 프로세스 (Phase A–E) |

판정: `python tools/ccr_validate_risk.py --draft <파일명>`

## Draft 작성 필수 필드

```yaml
---
제안팀: team1
제안일: 2026-04-15
영향팀: [team2, team4]      # 빈 배열 금지
변경 대상 파일:
  - docs/2. Development/2.5 Shared/Authentication.md
변경 유형: modify             # add | modify | remove | rename
변경 근거: ...
---
```

## Conductor 승격 워크플로우

사용자 "ccr promote" 트리거 → Phase A(Collection) → B(Planning) → C(Execution, 실제 docs 수정) → D(Clarification) → E(Finalize + commit).

상세: `../../CLAUDE.md` §"Conductor 승격 워크플로우"

## 도구

| 명령 | 목적 |
|------|------|
| `python tools/ccr_promote.py --validate-only` | pending/ draft 메타 JSON 출력 |
| `python tools/ccr_promote.py --complete <file> --number N --applied-files "<csv>"` | 편집 완료 후 마감 |
| `python tools/ccr_validate_risk.py --draft <file>` | 리스크 등급 판정 |
| `python tools/ccr_dashboard.py` | 현황 대시보드 |
