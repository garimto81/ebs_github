# 22종 게임 PRD 전면 재설계 계획

> **Date**: 2026-03-30
> **배경**: 현재 문서는 다이어그램 1개에 여러 개념을 압축. 게임 규칙은 복잡하므로 "한 다이어그램 = 한 개념" 원칙으로 전면 재설계 필요.

---

## 현재 문제점 분석

### "Community Card 계열이란?" 소개 (현재 3줄 + Mermaid 1개)

**문제**: 포커를 처음 접하는 사람에게 "비공개 카드", "공유 카드", "조합", "5장 선택"을 한 번에 던짐.

**재설계**: 4단계로 분리

| 단계 | 개념 1개 | 시각 자료 |
|:----:|---------|----------|
| 0-1 | "포커란 카드 게임입니다" — 52장 덱 소개 | 카드 덱 이미지 |
| 0-2 | "나만 보는 카드가 있습니다" — 비공개 개념 | face-down 카드 이미지 |
| 0-3 | "모두가 공유하는 카드도 있습니다" — 공유 개념 | 테이블 중앙 공개 카드 이미지 |
| 0-4 | "이 두 종류를 합쳐서 최고의 패를 만듭니다" | 비공개 + 공유 → 조합 다이어그램 |

### Hold'em Step 1 (현재 Mermaid 1개에 4개 화살표)

**문제**: 딜러→비공개2장, 딜러→테이블5장, 비공개→5장선택, 테이블→5장선택을 한 다이어그램에 압축.

**재설계**: Step 1은 "한 줄 요약"만. 상세는 Step 2~5에서 하나씩.

### Hold'em Step 4 (현재 Flop/Turn/River를 3개 서브섹션)

**현재 상태**: 이미 점진적 빌드업이 적용되어 있어 구조는 양호. 다만 각 스트릿에 시각 자료가 삽입되어 있으므로 큰 변경 불필요.

### Hold'em Step 5 핸드 랭킹 (현재 테이블 1개에 10행)

**문제**: 10등급을 한 번에 보여줌. 초보자에게 Royal Flush부터 High Card까지 10개를 한꺼번에 이해시키는 건 과부하.

**재설계**: 3그룹으로 나누어 단계적 소개

| 그룹 | 등급 | 설명 |
|------|------|------|
| **기본** | High Card, One Pair, Two Pair | "가장 흔한 패 3가지" |
| **중간** | Three of a Kind, Straight, Flush, Full House | "좀 더 특별한 패 4가지" |
| **희귀** | Four of a Kind, Straight Flush, Royal Flush | "거의 안 나오는 전설의 패 3가지" |

## 재설계 원칙

### 핵심: 한 시각 자료 = 한 개념

```
❌ 잘못: Mermaid 1개 안에 {딜러, 비공개카드, 공개카드, 5장선택}
✅ 올바름:
   시각 1: "딜러가 카드를 나눠줍니다" (딜러 이미지만)
   시각 2: "내 카드는 2장입니다" (face-down 2장 이미지만)
   시각 3: "테이블 가운데에도 카드가 깔립니다" (보드 이미지만)
   시각 4: "이 두 종류를 합칩니다" (조합 이미지)
```

### 정보 밀도 제한

| 위치 | 최대 정보량 |
|------|:---:|
| 한 다이어그램/이미지 | **개념 1개** |
| 한 단계(Step/Sub-step) | **개념 1개 + 시각 1개 + 캡션 1-2줄** |
| 한 테이블 | **행 최대 5개** (10행 → 3그룹으로 분리) |

### 시각 자료 우선순위

| 우선 | 시각 유형 | 용도 |
|:----:|----------|------|
| 1 | **스크린샷** (HTML 캡처) | 카드, 테이블, 비교 |
| 2 | **Mermaid flowchart** | 진행 흐름 (노드 4개 이하) |
| 3 | **테이블** | 비교, 랭킹 (5행 이하) |
| 4 | 텍스트 카드 표기 | fallback only |

## 재설계 구조: Hold'em 섹션 (가장 중요)

### Step 0. 포커란? (신규 — 완전 초보를 위한 도입)

```
0-1. 카드 덱
    "트럼프 카드 52장을 사용합니다"
    → 시각: 4가지 무늬(♠♥♦♣) + 13가지 숫자(2~A)

0-2. 게임의 목표
    "카드 조합으로 다른 플레이어를 이기는 것입니다"
    → 시각: 승자가 칩(돈)을 가져가는 모습

0-3. Community Card란?
    "내 카드 + 공유 카드 = 합쳐서 최고의 패를 만듭니다"
    → 시각: 비공개 [??][??] + 공유 5장 → 화살표 → "Best 5!"
```

### Step 1. Hold'em 한 줄 소개 (간소화)

```
"세계에서 가장 인기 있는 포커. 비공개 2장 + 공유 5장 = 최고 5장으로 승부"
→ 시각 없음 (한 줄 요약만)
```

### Step 2. 카드를 받습니다 (유지 + 보강)

```
2-1. "딜러가 카드를 나눠줍니다"
     → 시각: 딜러 위치 + 카드 배분 방향

2-2. "각자 2장을 받습니다. 뒤집어서 — 나만 봅니다"
     → 시각: cc-01 스크린샷 (3인 face-down)

2-3. "방송에서는? RFID가 시청자에게만 보여줍니다"
     → 시각: face-down(플레이어 시점) vs face-up(시청자 시점)
```

### Step 3. 베팅 (유지 — 현재 구조 양호)

```
3-1. "3가지 선택: Call / Raise / Fold"
     → 시각: 3가지 선택 비교 테이블

3-2. "이 선택은 매 단계마다 반복됩니다"
     → 시각: Mermaid 1개 (카드→베팅→카드→베팅→... 반복)
```

### Step 4. 보드 공개 (유지 — 점진적 빌드업 양호)

```
4-1. Flop — 3장
     → 시각: cc-02 스크린샷
     → "지금 총 5장 (내 2 + 보드 3)"

4-2. Turn — +1장
     → 시각: cc-03 스크린샷
     → "지금 총 6장"

4-3. River — 마지막 +1장
     → 시각: cc-04 스크린샷
     → "최종 7장 완성!"
```

### Step 5. 핸드 랭킹 (3그룹 분리 재설계)

```
5-1. "같은 숫자가 있으면 강합니다" (기본 3개)
     → 시각: High Card → One Pair → Two Pair
     → 캡션: "가장 흔한 패. 10판 중 7판은 이 패로 결정됩니다"

5-2. "연속이거나 같은 무늬면 더 강합니다" (중간 4개)
     → 시각: Three of a Kind → Straight → Flush → Full House
     → 캡션: "여기서부터 흥분이 시작됩니다"

5-3. "거의 안 나오는 전설의 패" (희귀 3개)
     → 시각: Four of a Kind → Straight Flush → Royal Flush
     → 캡션: "프로 선수도 평생 몇 번 못 봅니다"
```

### Step 6. 승부 (유지 + 보강)

```
6-1. 실전 예시 — 홀카드 공개
     → 시각: A의 [A♠][K♥] vs B의 [Q♦][Q♣]

6-2. 보드 + 조합 과정
     → 시각: 7장 중 5장 고르는 과정

6-3. 결과 비교
     → 시각: cc-05 승부 스크린샷
```
