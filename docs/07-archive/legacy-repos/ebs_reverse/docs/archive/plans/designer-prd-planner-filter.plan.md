# Plan: Designer PRD 기획자 영역 필터링

> **Phase 1 Designer PRD (v1.0.0)에서 개발자 영역을 제거하고, 기획자 관점으로 재작성**

---

## 배경

기존 Designer PRD v1.0.0은 기획자 73% / 개발자 27%로 혼합되어 있다. 기획자가 문서를 읽을 때 MCU, Serial, WebSocket, UID, DB 쿼리 등 불필요한 기술 세부사항에 노출되어 핵심 의사결정에 집중하기 어렵다.

## 문제 정의

| 문제 | 상세 |
|------|------|
| **기술 용어 노출** | MCU, Serial, WebSocket, UID, DB 등 개발자 전용 용어가 기획자 문서에 포함 |
| **개발자 관점 다이어그램** | 시스템 개념도, 데이터 흐름이 내부 구현 수준으로 작성됨 |
| **로드맵 기술 편향** | 구현 로드맵이 기술 작업 중심으로 기획자가 마일스톤을 파악하기 어려움 |
| **독자 혼란** | 기획자가 "이 문서가 나를 위한 것인가"를 판단하기 어려움 |

## 핵심 원칙

| 기획자 영역 (포함) | 개발자 영역 (제거) |
|-------------------|-------------------|
| 사용자가 **보는** 것 | 시스템이 **어떻게** 작동하는가 |
| 사용자가 **느끼는** 것 | 어떤 **프로토콜**을 사용하는가 |
| 사용자가 **경험하는** 것 | 어떤 **하드웨어 구성요소**가 필요한가 |
| **무엇**을, **왜**, **누가**, **언제** | 데이터가 내부적으로 어떻게 흐르는가 |
| 성공을 어떻게 **측정**하는가 | 성능을 어떻게 **달성**하는가 |

## 대폭 수정 대상
