# ADR-005: 렌더링 전략 (DirectX 11 → Web)

## 상태
미결 (결정 필요)

## 맥락

PokerGFX는 DirectX 11 기반 GPU 파이프라인으로 구동된다. GFX 오버레이를 실시간으로 비디오 출력에 합성하며, 초당 30/60프레임의 고성능 렌더링이 방송 품질에 직결된다. EBS는 이 시스템을 React/TypeScript 웹 기반으로 재구현하는 전략을 취하고 있다.

EBS의 GFX 오버레이 시스템은 다음 요소들로 구성된다.

- **Image 요소**: 플레이어 카드 이미지, 스폰서 로고, 배경 레이어 (41개 속성 필드)
- **Text 요소**: 플레이어 이름, 스택, 블라인드, 팟 크기 (52개 속성 필드)
- **Pip 요소**: 카드 수트 아이콘 (12개 속성 필드)
- **Border 요소**: UI 경계선, 강조 효과 (8개 속성 필드)

이 요소들은 Skin Editor(ADR-003)에서 정의한 스킨 데이터를 기반으로 렌더링된다. 렌더링 전략 선택은 성능, 구현 복잡도, 브라우저 호환성, Skin Editor 통합 방식에 직접 영향을 미친다. 또한 Preview Panel(M-02)의 실시간 GFX 프리뷰와 실제 방송 출력(O-01~O-05 파이프라인) 모두에 동일한 렌더링 엔진이 사용된다.

## 결정

**미결. 아래 선택지 중 하나를 결정해야 한다.**

| 선택지 | 특성 | 적합 조건 |
|--------|------|----------|
| (1) Canvas 2D | 단순 구현, WebGL보다 낮은 성능, 광범위한 브라우저 지원 | 60fps 이하 방송, 단순 오버레이 |
| (2) WebGL | 고성능 GPU 활용, 복잡한 구현, 셰이더 작성 필요 | 60fps 방송, 복잡한 애니메이션 효과 |
| (3) WebGPU | 최신 GPU API, 최고 성능, 브라우저 호환성 제한 (Chrome 113+ 필요) | 미래 지향, 호환성 이슈 감수 가능 시 |
| (4) Rive/Lottie | 애니메이션 특화, 디자이너 친화, 정적 GFX에는 과잉 | 카드 공개 등 애니메이션 중심 연출 |

## 근거

현재 이 결정이 미결인 이유:

| 미결 사유 | 내용 |
|----------|------|
| 성능 요건 미확정 | 동시 플레이어 수(최대 10명), 애니메이션 복잡도, 목표 프레임레이트가 확정되지 않음 |
| 플랫폼 전략 미확정 | 브라우저 기반 단독 운영인지, Electron 래핑인지에 따라 WebGPU 호환성 영향이 달라짐 |
| Skin Editor 통합 미확정 | ADR-003(Skin Editor v2.0 Defer)으로 인해 v1.0은 기본 스킨 1종만 렌더링하면 됨. v2.0에서 Skin Editor가 활성화될 때의 렌더링 요건을 고려해야 함 |

**임시 방향**: v1.0에서는 Canvas 2D로 시작하고 v2.0에서 WebGL로 마이그레이션하는 2단계 접근이 유력하나, 마이그레이션 비용 분석이 필요하다.

## 영향

이 결정이 내려지기 전까지 블로킹되는 구현:

- **Preview Panel (M-02)**: 실시간 GFX 프리뷰 렌더링 구현 착수 불가
- **GFX 오버레이 전체**: Image(41필드), Text(52필드), Pip(12필드), Border(8필드) 렌더링 구현 착수 불가
- **Transition 애니메이션 (G-17, G-18)**: 등장/퇴장 애니메이션 구현 방식에 영향
- **Skin Editor (ADR-003)**: v2.0에서 Skin Editor 개발 시 렌더링 엔진과의 통합 설계에 영향

렌더링 전략 확정 후 이 ADR을 업데이트하고 블로킹된 구현을 착수한다.

## 관련 요소

- M-02 (Preview Panel) — 렌더링 엔진 직접 사용
- O-01~O-05 (Live 출력 파이프라인) — 렌더링 출력 경로
- G-17 (Transition In) — 애니메이션 렌더링 의존
- G-18 (Transition Out) — 애니메이션 렌더링 의존
- SK-01~SK-26 (Skin Editor, ADR-003) — v2.0 통합 시 렌더링 엔진 연계
- GFX Image/Text/Pip/Border 필드 (113개 속성) — 렌더링 구현 대상 전체
