# [B-333] Overview §1.2.1 game_class 에 Foundation 12/7/3 분류 + 22 variant 매핑표 신설 (P1)

## 배경

Foundation §6.1 은 22 종 포커를 다음 3 계열로 분류:

| 계열 | 종수 | 특징 |
|------|:----:|------|
| 공유 카드 존재 | **12** | Flop 계열 (Hold'em, Omaha, Short Deck 등) |
| 카드 교환 가능 | **7** | Draw 계열 (Five Card Draw, 2-7 Triple Draw, Badugi 등) |
| 일부만 공개 | **3** | Stud 계열 (Stud, Razz, Stud Hi-Lo) |

그러나 team3 `Behavioral_Specs/Overview.md §1.2.1 game_class` 는 현재 `flop` 한 값만 테이블에 등록하고 나머지는 "Tier 3~4 문서 참조" 로 넘기고 있다. Foundation 확정치 12/7/3 이 정확히 어느 variant 에 매핑되는지 **단일 정본 표** 가 없다.

## 수정 대상

### `Behavioral_Specs/Overview.md`
1. §1.2.1 `game_class` 표를 3 값 완성:
   | 값 | 이름 | 계열 특징 | Foundation 종수 |
   |:--:|------|----------|:---------------:|
   | 0 | flop | 커뮤니티 카드 5장 | 12 |
   | 1 | draw | 카드 교환 | 7 |
   | 2 | stud | 일부 공개 | 3 |

2. §1.1 `game` enum 표를 22 값 완전 기재 (현재 `0=holdem` 만). Flop_Variants.md / Draw_Games.md / Stud_Games.md 와 대조 완결

3. §1.2.4 `event_game_type` 매핑 수정/검증: 현재 Holdem(0-3 = 4) + Omaha(4-11 = 8) = 12 Flop ✓ 확인. Draw/Stud 매핑이 7/3 과 정확히 일치하는지 verify

### 검증 산출물
- `tests/variant_catalog_test.dart` — 22 variant 가 각 계열별로 12/7/3 으로 분류되는지 assert (Pure Dart test)
- `dart analyze` 0 errors 유지
