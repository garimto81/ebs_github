## 토큰 검증 가드 (GAP-L-001)

> **가드 조건**: 세션 복원은 반드시 **토큰 유효성 검증 이후**에만 실행한다.
>
> - `GET /auth/session` 응답이 200 OK일 때만 세션 복원 다이얼로그를 표시한다.
> - 401/403 응답 시 세션 복원을 건너뛰고 로그인 화면으로 리다이렉트한다.
> - localStorage에 토큰이 존재하더라도 **서버 검증 없이** 세션 복원을 진행해서는 안 된다.

### 검증 흐름

| Step | 동작 | 결과 |
|:----:|------|------|
| 1 | 앱 시작 / 재접속 | localStorage 토큰 확인 |
| 2 | 토큰 있음 → `GET /auth/session` 호출 | — |
| 3 | 200 OK | `Session_Init` 데이터 복원, 복원 다이얼로그 표시 |
| 4 | 401/403 | 토큰 폐기, 로그인 화면(`Form.md`)으로 리다이렉트 |
| 5 | 5xx / 타임아웃 | 재시도 UI + 오프라인 모드 폴백 |

## 로그인 실패 처리

| 시나리오 | 응답 | UI 처리 |
|----------|------|---------|
| 잘못된 email/password | 401 | "이메일 또는 비밀번호가 올바르지 않습니다" inline 에러 |
| 계정 잠금 (연속 실패 N회) | 423 | "계정이 잠겼습니다. 관리자에게 문의하세요" 모달 + lock duration 표시 |
| 2FA 코드 불일치 | 401 | "인증 코드가 일치하지 않습니다" + 재입력 focus |
| 2FA 코드 만료 | 401 | "인증 코드가 만료되었습니다. 다시 받으세요" + resend 버튼 |
| 네트워크 오류 | — | "네트워크에 연결할 수 없습니다" + 재시도 버튼 |

> 상세 연속 실패 lockout 정책: `../../2.5 Shared/Authentication.md` §Lockout Policy.
