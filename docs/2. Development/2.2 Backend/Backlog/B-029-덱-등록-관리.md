## DRIFT-50.2 dup-uid 422 정합 (2026-05-12, S7 Cycle 5, issue #282)

**누적 출처**: Cycle 2 PR #248 + Cycle 4 PR #277 = 2 cycle HIGH 미해소.

### 증상

`integration-tests/scenarios/50-rfid-deck-register.http` step 50.2 가 **422 DUPLICATE_UID** 명세하지만 BO `team2-backend/src/routers/decks.py` 는 **409 RFID_UID_DUPLICATE** 반환. 동일 UID 카드를 다른 cardCode 로 재등록 시 422 가 와야 하는데 실제 코드 status = 409.

### 결정

`.http` SSOT 정합 — 코드를 422/`DUPLICATE_UID` 로 변경 (RFC 상 409 가 더 정확하지만 contract 문서 + 이슈 SSOT 가 422 명시. Type D drift 해소 원칙: SSOT = 진실).

### 변경

| 위치 | 변경 |
|------|------|
| `decks.py register_card` | dup-uid 시 409→422, code RFID_UID_DUPLICATE→DUPLICATE_UID, response 에 `uid` 필드 추가 |
| `decks.py import_deck` | cross-deck dup 409→422, 신규 **intra-payload dup uid 검증** (dict[code→uid] 의 동일-값 다른-키 함정) |
| `decks.py replace_card` | cross-uid 충돌 시 409→422, code 정합 |

| `tests/test_decks_inmemory.py` | 기존 cross-deck/cross-import 409→422 갱신 + 신규 3 테스트 (same_deck_dup / intra_payload / replace_dup) |
