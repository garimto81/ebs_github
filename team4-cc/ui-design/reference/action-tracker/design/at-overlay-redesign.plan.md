## 위험 요소 (Risks)

1. **대형 문서 토큰 초과**: 559줄 → 재설계 후 700줄+ 예상. Map-Reduce 청킹 필수 (12-large-document-protocol.md 준수)
2. **4장 전체 배치도 재설계 복잡도**: 기존 10인 고정 배치 → 액티브 플레이어만 표시 + 가장자리 배치 + 커스터마이징. 설계 일관성 유지 필요
3. **3장-4장 간 데이터 흐름 정합성**: AT 좌석 레이아웃 변경(P1~P9, 딜러 제거)과 오버레이 Player Graphic 배치가 연동되어야 함. 불일치 시 구현 단계에서 혼선
4. **템플릿 시스템 설계 범위 확대**: "풀 커스텀 가능한 템플릿"이 기술 아키텍처(v2.0)의 NodeCG/Rive 렌더링 파이프라인과 호환되는지 검증 필요

### Edge Cases
- **1인 플레이 시 오버레이**: 액티브 플레이어가 1명일 때 레이아웃 붕괴 방지 (최소 배치 규칙 필요)
- **10인 풀 테이블에서 모든 플레이어 액티브**: 가장자리 배치로 10인 Player Graphic이 모두 표시될 때 영상 영역 침범 여부
- **세로(9:16) 모드 오버레이**: 가로 기준 가장자리 배치가 세로에서 유효한지

## 태스크 목록 (Tasks)

### Task 1: 3.1 메인 레이아웃 재설계

**설명**: AT 메인 레이아웃 ASCII 다이어그램과 설명 전면 교체
- 딜러 포지션(D) 제거
- 좌석 번호를 P1부터 시작하도록 재구성 (기존 P6 → P1)
- 최대 9 플레이어 배치 (10인 → 9인)
- ASCII 다이어그램 재작성

**수행 방법**: `EBS-UI-Design-v3.prd.md` 223~241줄 (3.1 메인 레이아웃) 교체

**Acceptance Criteria**:
- P1~P9까지 9좌석 배치 (딜러 포지션 없음)
- ASCII 다이어그램에 딜러(D) 좌석 부재
- P1이 첫 번째 좌석 (기존 P6 위치)

### Task 2: 3.2 좌석 레이아웃 업데이트

**설명**: 테이블 형태를 템플릿 기반으로 변경
- 테이블 형태: 타원형, 육각형, 반원형 등 템플릿 선택 가능
- 플레이어 배치: 고정 좌표 → 그리드 기반 배치
- 최대 10인 세팅 가능 (AT 내부 세팅용)

**수행 방법**: `EBS-UI-Design-v3.prd.md` 243~262줄 (3.2 좌석 레이아웃) 교체

**Acceptance Criteria**:
- 테이블 형태 템플릿 목록 (최소 3종: 타원형, 육각형, 반원형) 정의
- 그리드 기반 배치 명세 포함
- 최대 10인 세팅 명시
- 각 템플릿별 좌석 배치 ASCII 다이어그램 포함

### Task 3: 3.3~3.5 액션 패널/베팅/상태전환 수정

**설명**: 좌석 수 변경(9인)에 따른 연쇄 수정
- 액션 패널: 좌석 참조 업데이트
- 베팅 입력: 변경 없음 (독립적)
- 화면 상태 전환: 9인 기준으로 정보 밀도 재조정

**수행 방법**: `EBS-UI-Design-v3.prd.md` 263~296줄 수정

**Acceptance Criteria**:
- 10인 → 9인 참조 전부 갱신
- 기능적 내용은 유지 (액션 버튼 구성, 베팅 프리셋)

### Task 4: 4.1 전체 배치도 전면 재설계

**설명**: 오버레이 전체 배치 철학 변경 — "가장자리에 덧입히는 부가 그래픽"

핵심 변경:
- 오버레이 = 방송 영상 위에 덧입히는 부가 요소 → 가장자리 배치
- 메인 영상(방송 카메라)이 화면 중심을 차지
- 전체 플레이어 표시 X → **액티브 플레이어(폴드 안 한)만 표시**
- 기본 형태: **3플레이어** 플레이 형태
- 배치 커스터마이징: 좌정렬/우정렬/하단/상단 변경 가능
- 픽셀 단위 크기/위치 설정
- 커스터마이징 변형값 템플릿 제공
- HTML 기반 설계

**수행 방법**: `EBS-UI-Design-v3.prd.md` 321~347줄 (4.1 전체 배치도) 전면 교체

**Acceptance Criteria**:
- 기본 배치 ASCII: 3명 액티브 플레이어 + 가장자리 배치 (화면 중앙 비움)
- 변형 템플릿 최소 3종 (좌정렬, 우정렬, 하단 집중) ASCII 다이어그램
- 배치 커스터마이징 속성 테이블 (x, y, width, height, alignment)
- 액티브 플레이어만 표시 규칙 명세
- HTML 기반 렌더링 명시

### Task 5: 4.2 Player Graphic 템플릿화

**설명**: Player Graphic을 풀 커스텀 가능한 템플릿 형태로 재설계

변경:
- 고정 레이아웃 → HTML 템플릿 기반 커스텀
- 서브 컴포넌트 배치/크기/표시여부 모두 설정 가능
- 기본 템플릿 + 변형 템플릿 제공
- 픽셀 단위 크기 설정

**수행 방법**: `EBS-UI-Design-v3.prd.md` 349~388줄 (4.2 Player Graphic) 교체

**Acceptance Criteria**:
- HTML 템플릿 구조 명세 (어떤 요소가 커스텀 가능한지)
- 기본 템플릿 + 최소 2개 변형 템플릿 정의
- 커스터마이징 가능 속성 테이블
- Glassmorphism 스타일 유지 (BM-3 패턴)

### Task 6: 4.3 Board Graphic 템플릿화

**설명**: Board Graphic을 풀 커스텀 가능한 템플릿 형태로 재설계

**수행 방법**: `EBS-UI-Design-v3.prd.md` 389~412줄 (4.3 Board Graphic) 교체

**Acceptance Criteria**:
- HTML 템플릿 구조 명세
- 커스터마이징 가능 속성 테이블
- 기본 + 변형 템플릿 정의

### Task 7: 4.4~4.7 보조 오버레이 전면 설계

**설명**: 현재 일부 미설계된 보조 오버레이 (Blinds, Field, Leaderboard, Ticker, Strip, Outs, Cards, 게임별 카드 수)에 HTML 템플릿 제공

변경:
- 4.4 Blinds Graphic → HTML 템플릿 + 커스터마이징
- 4.5 보조 오버레이 6종 → 각각 HTML 템플릿 상세 설계
- 4.6 Cards 에셋 → 변경 최소 (에셋 규격)
- 4.7 게임별 카드 수 → 변경 최소

**수행 방법**: `EBS-UI-Design-v3.prd.md` 414~463줄 (4.4~4.7) 교체/확장

**Acceptance Criteria**:
- 모든 오버레이에 HTML 템플릿 구조 명세
- 각 오버레이별 커스터마이징 속성 테이블
- 변형 템플릿 최소 1종씩

### Task 8: 5장 화면 전환 수정

**설명**: 배치 변경에 따른 5.1~5.3 연쇄 수정

변경:
- 5.1: 액티브 플레이어만 표시 규칙 반영 (폴드 시 오버레이에서 제거)
- 5.2: 이벤트 강조 — 가장자리 배치에 맞춘 연출 위치 조정
- 5.3: Live vs Delayed 모드 — 변경 최소

**수행 방법**: `EBS-UI-Design-v3.prd.md` 464~505줄 수정

**Acceptance Criteria**:
- 폴드 플레이어 오버레이 제거 규칙 명시 (기존: opacity 0.4 → 신규: 제거)
- 액티브 플레이어 수 변동에 따른 배치 자동 조정 명세

