# 버저닝 규칙

## 두 가지 버전

이 저장소에는 **문서 버전**과 **릴리즈 버전**, 두 가지 버전이 존재합니다.

### 문서 버전

- 각 개별 문서가 자체적으로 갖는 버전
- 문서 내부에 Version과 변경 히스토리를 기록
- 예: `PRD-GAME-01-flop-games.md` → v1.0.0

### 릴리즈 버전

- 여러 문서를 종합한 하나의 릴리즈 단위
- `release/vX.Y.Z` 브랜치로 관리
- 예: `release/v0.2.0` = 문서 A의 v1 + 문서 B의 v3 + 문서 C의 v2

## Semantic Versioning

`vMAJOR.MINOR.PATCH` 형식을 사용합니다.

### 릴리즈 버전 기준

| 구분 | 기준 | 예시 |
|------|------|------|
| **Major** | 기존 구조가 깨지는 변경. 새로운 게임 계열 추가, 시스템 전면 재설계 등 | Draw 계열 전체 추가, 베팅 시스템 전면 재설계 |
| **Minor** | 새로운 게임/기능 추가, 기존 스펙의 확장 | Omaha 게임 추가, PoC → MVP 스펙 확장 |
| **Patch** | working 브랜치 내 수정 — 오타, 피드백 반영, 명확화 | 규칙 설명 보완, AC 수정, 백로그 조정 |

### 릴리즈 브랜치 규칙

- **킥오프는 Minor 이상의 변경이어야 한다**
  - `working/v0.2.0` (O) — 새로운 기능/게임 추가
  - `working/v0.1.1` (X) — Patch만으로는 킥오프하지 않음
- **working 브랜치 내에서 버전을 올릴 수 있으며, Patch로 한다**
  - `working/v0.2.0` 에서 피드백 반영 → v0.2.1, v0.2.2, ...

### 문서 버전 기준

각 문서 내부에서 자체적으로 버전을 관리합니다.

| 구분 | 기준 |
|------|------|
| **Major** | 문서 구조 변경, 대규모 내용 재작성 |
| **Minor** | 새로운 섹션 추가, 기존 내용의 의미 있는 확장 |
| **Patch** | 오타 수정, 표현 개선, 사소한 보완 |

## 문서 내부 버전 표기

각 문서의 상단에 다음 형식으로 기록합니다:

```markdown
> **Version**: 1.2.0
> **Date**: 2026-04-02

## 변경 히스토리

| 버전 | 날짜 | 변경 내용 |
|------|------|----------|
| 1.2.0 | 2026-04-02 | Omaha Hi-Lo 섹션 추가 |
| 1.1.0 | 2026-03-30 | Short Deck 규칙 보완 |
| 1.0.0 | 2026-03-28 | 초기 작성 |
```

## 릴리즈와 문서 버전의 관계

릴리즈는 특정 시점의 문서 버전들을 묶은 것입니다.

```
release/v0.2.0
  ├── PRD-GAME-01-flop-games.md        → v1.2.0
  ├── PRD-GAME-04-betting-system.md    → v2.0.0
  ├── poc-scenario.prd.md              → v3.1.0
  └── backlogs/                        → 이 버전들 기준으로 생성
```

릴리즈 브랜치의 `backlogs/README.md`에 포함된 문서 버전을 명시합니다.
